B. UI LAYER (Design System)

Mục tiêu:

Xây nền UI độc lập với business logic, tái sử dụng được, kiểm soát consistency toàn hệ thống.

Chia thành 3 tầng:

B1. Foundation (Atomic UI)
B2. Composite UI (Reusable Blocks)
B3. Domain UI (Business-aware)

B3. DOMAIN UI (Business-aware Components)

Tại đây mới gắn với exam system.

1️⃣ ENUM / STATUS MAPPING COMPONENTS (Foundation của B3)

Encapsulate mapping từ DB enum → UI.

1. role-badge.tsx

Input:

role: "admin" | "teacher" | "student"

Output:

color
label
icon (optional)
2. exam-status-badge.tsx

Mapping từ bảng exams.status

Ví dụ:

type ExamStatus =
  | "draft"
  | "published"
  | "archived"
3. exam-state-badge.tsx

Runtime state của 1 exam instance:

type ExamState =
  | "not_started"
  | "in_progress"
  | "submitted"
  | "graded"

Khác với exam.status.

4. difficulty-badge.tsx
type Difficulty =
  | "easy"
  | "medium"
  | "hard"
5. question-type-badge.tsx

Mapping theo engine bạn đã định nghĩa:

MCQ_SINGLE
TEXT_INPUT
PASSAGE_MCQ
PASSAGE_TEXT
REORDER
TRUE_FALSE
AUDIO_MCQ
AUDIO_TEXT
ESSAY

Chuẩn hoá luôn ExamStatus
Chuẩn hoá ExamState
Chuẩn hoá Difficulty


6. passage-type-badge.tsx

Vì system enforce 1 passage = 1 question_type.

👉 Đây là lớp quan trọng nhất.

Toàn bộ visual consistency của system phụ thuộc vào đây.
